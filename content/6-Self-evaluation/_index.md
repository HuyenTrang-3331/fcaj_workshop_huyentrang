Throughout my participation in the First Cloud Journey program from April 17, 2026, to July 10, 2026, I had the opportunity to learn, practice, and systematize a lot of important knowledge about AWS Cloud. The program helped me access foundational service groups like IAM, VPC, EC2, S3, RDS, Database Migration, Security, Monitoring, and Cost Optimization.

Through learning modules and hands-on labs, I not only understood the theory of each AWS service but also got to directly interact with the AWS Console, configure resources, check results, record errors, and clean up resources after completing labs. This helped me develop self-learning skills, architecture analysis thinking, debugging ability, and the habit of cost management when using cloud services.

In terms of learning attitude, I always tried to complete content week by week, actively took notes on important knowledge, reviewed results after each lab, and synthesized them into reports. Although there are still some areas for improvement, overall the learning process has given me a better foundation to continue developing in the Cloud Computing field.

### Strengths
* Ability to systematize AWS knowledge by service groups like Compute, Storage, Networking, Database, Security, and Monitoring.
* Understand the role of core services like IAM, VPC, EC2, S3, RDS, CloudWatch, and CloudTrail in a basic AWS architecture.
* Know how to perform labs according to the process: read requirements, configure resources, check results, note errors, and clean up resources.
* Better thinking in analyzing AWS architecture diagrams, correctly identifying service positions and recognizing some common design errors.
* Clearer awareness of account security, Least Privilege principle, MFA, IAM Role, and risks of granting overly broad permissions.
* Know how to pay attention to costs when using AWS, especially cost-prone resources like EC2, RDS, NAT Gateway, Load Balancer, CloudWatch Logs, and S3.
* Improved note-taking, synthesizing learning content, and presenting technical reports.

### Areas for Improvement
* Need more practice designing complete AWS architectures for real systems, not just stopping at individual services.
* Need to improve ability to read official AWS documentation in English to understand each service more deeply.
* Need more practice with AWS CLI, CloudFormation, or Terraform to get familiar with Infrastructure as Code.
* Need to train more troubleshooting skills when encountering complex errors related to network, permissions, logs, or cost.
* Need better time management for learning so that note-taking, practice, and report synthesis are more consistent.
* Need to continue improving presentation skills, explaining architectures, and expressing technical issues concisely and clearly.

### Improvement Orientation in the Future

In the future, I will continue to strengthen AWS Cloud knowledge by practicing more real architecture models like web applications, serverless applications, data pipelines, and monitoring systems. Additionally, I will also focus on deeper learning about security, cost optimization, infrastructure automation, and application deployment on AWS.

I also set goals to improve architecture analysis, reading technical documents, debugging, and presenting solutions. These are important skills to apply AWS knowledge to real projects in study and future work.

### Conclusion

Overall, the process of participating in First Cloud Journey has helped me make clear progress in AWS knowledge, lab practical skills, and cloud system deployment thinking. I have better understood the role of each service group, know how to link theory with practice, and formed the habit of checking security, costs, logs, and resources after each lab.

Although there are still some areas for improvement, I evaluate this as a very useful learning phase that helped me build a good foundation to continue developing in Cloud Computing, DevOps, Backend, or System Administration in the future.
