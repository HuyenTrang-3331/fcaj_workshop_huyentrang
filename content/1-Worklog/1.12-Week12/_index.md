### Week 12 Objectives:

* Learn AWS services for monitoring, logging, security, and risk detection.
* Review the role of Amazon CloudWatch, AWS CloudTrail, AWS Config, Amazon GuardDuty, and AWS Security Hub.
* Learn how to monitor system activity through logs, metrics, and alarms.
* Learn how to detect unusual behavior and security risks on AWS.
* Practice log review, alert assessment, and incident response recommendation skills.
* Synthesize knowledge for final stage report and presentation.

### Week 12 Achievements:

* Understand the role of monitoring and logging in AWS system operation.
* Master the role of Amazon CloudWatch in resource monitoring.
* Understand the main components of CloudWatch, including:
  * Metrics.
  * Logs.
  * Alarms.
  * Dashboards.
* Know how CloudWatch supports monitoring the status of EC2, RDS, Lambda, and other AWS services.
* Understand the role of AWS CloudTrail in recording API activity on the AWS account.
* Can differentiate CloudWatch and CloudTrail:
  * CloudWatch is for monitoring performance, application logs, and alerts.
  * CloudTrail is for recording user actions, API calls, and account administration activity.
* Understand the role of Amazon GuardDuty in detecting unusual behavior or security risks.
* Understand the role of AWS Security Hub in aggregating findings from many security services.
* Master basic concepts such as:
  * Finding.
  * Severity.
  * Remediation.
  * Threat Detection.
* Understand the role of AWS Config in tracking resource configuration and compliance checking.
* Know how to relate monitoring, logging, and security into real-world system operation.
* Practice error investigation thinking based on logs and alerts.
* Know how to recommend initial response direction when there are security or performance alerts.
* Complete study notes about CloudWatch, CloudTrail, GuardDuty, Security Hub, and AWS Config.
