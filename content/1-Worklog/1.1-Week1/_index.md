### Week 1 Objectives:

- Connect and get acquainted with members of First Cloud AI Journey.
- Understand the overview of AWS, Cloud Computing, and the role of cloud computing in enterprises.
- Master basic AWS service groups such as Compute, Storage, Networking, Database, Security, and Management Tools.
- Get familiar with AWS Free Tier, AWS Management Console, and AWS CLI.
- Learn how to use AWS CLI to interact with AWS resources at a basic level.
- Understand the general AWS architecture diagram and how to represent key components in a cloud system.

### Week 1 Achievements:

- Understood the overview of AWS and its role in the cloud computing model.
- Mastered the basic benefits of Cloud Computing in system deployment, including:
    - Flexible scalability.
    - Rapid deployment.
    - Reduced initial infrastructure investment costs.
    - Centralized resource management.
    - Support for multiple service models.
- Mastered basic AWS service groups such as:
    - Compute.
    - Storage.
    - Networking.
    - Database.
    - Security.
    - Management Tools.
- Understood the learning objectives of the First Cloud AI Journey program and contents to be completed in the first week.
- Learned about AWS Free Tier and understood the significance of using a Free Tier account during learning and practice.
- Created an AWS Free Tier account for learning and practicing basic AWS labs.
- Became familiar with AWS Management Console and learned how to:
    - Search for AWS services.
    - Access each service.
    - Observe resource management interface.
    - Monitor basic account information.
    - Choose the appropriate region when using AWS services.
- Installed and configured AWS CLI on the computer, including:
    - Access Key.
    - Secret Key.
    - Default Region.
    - Output format.
- Used AWS CLI to perform basic operations such as:
    - Check configuration information.
    - View region list.
    - Check EC2 service information.
    - Create and manage key pairs.
    - Observe active resources.
- Understood the role of AWS CLI in managing AWS resources via command line, supporting faster operations compared to the web interface in some cases.
- Understood common components in general AWS architecture diagrams such as:
    - User/Client.
    - Region.
    - Availability Zone.
    - VPC.
    - Public Subnet / Private Subnet.
    - Internet Gateway.
    - NAT Gateway.
    - EC2.
    - Load Balancer.
    - Auto Scaling Group.
    - RDS.
    - S3.
    - IAM.
    - CloudWatch.
- Practiced sketching a general AWS architecture diagram, showing user access through various layers:
    - Network.
    - Compute.
    - Database.
    - Storage.
    - Security.
    - Monitoring.
- Capable of describing a basic AWS architecture at a high level, including access flow and roles of main service groups.
- Capable of combining knowledge of AWS Management Console, AWS CLI, and architecture diagrams to describe basic AWS resources.
- Completed all learning and practice contents in Week 1 of First Cloud AI Journey.
